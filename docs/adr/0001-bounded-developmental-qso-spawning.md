# ADR-0001: Bounded developmental QSO spawning

## Status

Accepted for initial implementation.

## Context

Seeker data will contain objectives, needs, curiosities, hypothetical scenarios, evidence, priors, and synthetic or analytic judgments. The system needs to transform that material into new QSO genomes and runtime instances while preserving provenance, reversibility, resource boundaries, and falsifiable evaluation.

## Decision

1. Genome IDs are derived from canonicalized handoff content and parent lineage.
2. Runtime identity is separate from genome identity.
3. Every QSO receives explicit compute, memory, curiosity, replication, and freeze limits.
4. Protected ethical constraints are immutable within the initial runtime.
5. Development is measured through operational indicators such as coherence, uncertainty calibration, self-correction, transfer, ethical consistency, and feedback-loop integration.
6. The implementation does not equate those indicators with subjective sentience or consciousness.
7. Handoff material distinguishes evidence from reports, inferences, and hypotheses.

## Consequences

The first release favors auditability and controlled experiments over unconstrained self-modification. Later work may introduce signed genome registries, mutation operators, inter-QSO protocols, sandboxed tool use, and richer evaluation harnesses without weakening the initial invariants.
