# Experimental QSO Genome and Spawning Runtime

This experimental module prepares QSO-SEEKER for structured data handoff without changing the repository's existing hostile-input security boundary or release candidate.

## Capabilities

- Strict structured handoff parsing for objectives, needs, curiosities, scenarios, priors, evidence, and synthetic or analytic judgments.
- Deterministic genome identifiers and explicit lineage fields.
- Immutable ethical defaults protecting human autonomy, truthfulness, reversibility, and bounded replication.
- Population and step limits with automatic freezing.
- Tamper-evident HMAC manifests for controlled transfer to QSO-GENOMES.
- Append-only hash-linked collection events.
- Fixture-only batch execution with no network, credentials, subprocess execution, or autonomous external mutation.

## Research boundary

Sentience, ethical autonomy, quantum cognition, and cybernetic consciousness remain research hypotheses. This module creates auditable operational structures and does not claim subjective experience or consciousness.

## Intended handoff

The first collected dataset should be converted into JSON objects accepted by `Handoff.parse`, run through `run_batch`, and reviewed as generated manifests and ledger entries before any downstream publication or wider execution.

## Security posture

The module is isolated under `experimental/qso_spawn`. It does not modify existing retrieval, sanitizer, workflow, security-envelope, network-policy, or release files. Future network or repository publication adapters require separate review and explicit authorization.
